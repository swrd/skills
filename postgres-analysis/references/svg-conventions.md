# SVG 生成规范

本文件定义 PostgreSQL 分析中所有 SVG 图表的生成标准。

## 通用规范

### 强制规则

- **禁止 `<style>` 块**：所有样式必须内联写在元素属性中
- **数据来源注释**：涉及源码参数时，用 XML 注释标注来源文件和行号
- **字体规范**：不使用中文专用字体名，使用通用 sans-serif / monospace

### 安全边距

```
viewBox 设置：预留 10-15% 安全边距
  - 左右边距: 最少 30px
  - 上下边距: 最少 20px
  - 元素 x 坐标不超过 (viewBox宽度 - 30)
  - 右侧文本使用 text-anchor="end"
  - 左侧文本使用 text-anchor="start"
```

### 字体大小参考

| 用途 | 字号 | 字体 |
|------|------|------|
| 主标题 | 16-18pt | sans-serif, bold |
| 副标题 | 12-14pt | sans-serif, bold |
| 正文/标签 | 10-11pt | monospace |
| 注释/说明 | 8-9pt | sans-serif |

### 调色板

| 用途 | 填充色 | 描边色 |
|------|--------|--------|
| 背景 | `#f8fafc` | `#e2e8f0` |

| 次节点 | `#dcfce7` (绿) | `#22c55e` |

| 文本 | `#1e293b` | — |
| 浅色文本 | `#64748b` | — |

## 图表类型规范

### 1. 流程图

#### 布局原则
- **垂直主线**：主流程沿单一垂直轴展开
- **分支方向**：Yes 向右，No 向左或向下
- **节点间距**：垂直 ≥40px，水平分支间距 ≥80px

#### 节点尺寸

| 节点类型 | 形状 | 推荐尺寸 |
|----------|------|----------|
| 开始/结束 | 圆角矩形 (rx≥18) | 100×36 |
| 处理步骤 | 矩形 (rx=6) | 140-160×40 |
| 判断节点 | 菱形 | 宽80-90, 高80-90 |
| 终止节点 | 圆角矩形 (rx≥20) | 100-130×40 |

#### viewBox 预估
```
viewBox高度 = 顶部区域 + (节点数 × 90) + 底部边距
每个判断节点额外 +50px（分支空间）
底部边距至少 30px
```

#### 边界检查
```
最右元素 x + width ≤ viewBox宽度 - 30
最底元素 y + height ≤ viewBox高度 - 20
```

#### 连线规范
- 使用折线 `<path>` 避免穿过节点
- 判断节点四个顶点预留连接空间
- 箭头使用 `<marker>` 定义

#### 跨分支碰撞预防（强制）

从同一判断节点分出的多条分支，其操作节点可能共用相同的 x 列区域，导致纵向重叠。

**规则 1：分支列化**
- 从同一判断节点分出的多条分支，各分支的操作节点尽量分配到不同 x 列
- 若共用 x 列，必须保证纵向间距充足

**规则 2：纵向深度预估**
布局前先计算每个分支从分叉点到最远节点的纵向深度：
```
右分支深度 = Σ(节点高度 + 节点间距)
下游分支起始 y = 分叉点 y + max(下分支最小间距, 右分支总深度) + 安全间距(≥40px)
```

**规则 3：跨分支碰撞检查**
在阶段 1 布局规划的节点边界表中，对所有属于不同分支的节点对执行：
```
若 x 范围有交集 (x1_a < x2_b && x2_a > x1_b):
    则 y 间距必须 ≥ 40px
    否则 → 调整下游分支的起始 y 坐标
```

**规则 4：绕行连线**
当连线必须穿过另一分支的节点区域时：
- 使用折线 `<path>` 先水平偏移到无冲突的 x 位置
- 再垂直向下到达汇合线
- 示例：`(482,240) → (482,252) → (575,252) → (575,370)` 绕过右侧节点

**典型反例**（已修复）：
```
D1 "是" → P1 → D2 (y=180-240, x=419-545)
D1 "否" → D3 → P2 (y=230-270, x=408-556)
                    ↑ D2 底部 y=240 与 P2 顶部 y=230 重叠
修复: D3/P2 下移至 y=270+ ，D2 "否" 连线绕行至 x=575 避开 P2
```

### 2. 架构图

#### 布局原则
- 分层布局：每层一个水平带
- 层间使用单向箭头
- 同层元素保持 ≥60px 间距

#### 模块着色

| PostgreSQL 模块 | 颜色 |
|----------------|------|
| parser/rewrite | `#dbeafe` (浅蓝) |
| optimizer | `#e0e7ff` (靛蓝) |
| executor | `#dcfce7` (浅绿) |
| storage/buffer | `#fef3c7` (浅黄) |
| storage/lmgr | `#fee2e2` (浅红) |
| access/transam | `#f3e8ff` (浅紫) |
| utils | `#f1f5f9` (浅灰) |
| postmaster | `#cffafe` (浅青) |

### 3. 位字段图（数据结构位布局）

#### 宽度计算
```
每位宽度 = 可用宽度 / 总位数
位号标注在字段边界上方 ±4px
```

#### 对齐检查
```
位号 x 坐标 = 字段起始/结束 x 坐标 ± 4px
```

#### 标注规范
- 字段名居中于字段区域
- 保留/未使用字段用灰色填充
- 位号在上方，字段名在中间，类型/值在下方

### 4. 结构体内存布局图（新增）

用于可视化 C 结构体的内存布局，包括偏移、大小、对齐和填充。

#### 布局原则
- **垂直堆叠**：每个字段为一个水平矩形条
- **宽度**：按字节比例（如 8 字节 = 基准宽度，按比例缩放）
- **基准宽度**：使用最大成员的字节数作为满宽度

#### 元素规范

| 元素 | 样式 |
|------|------|
| 数据字段 | 实色填充（按模块着色），矩形 |
| Padding 字段 | 斜线填充或浅灰色，标注 "(padding)" |
| 偏移标注 | 左侧，十六进制（如 0x00, 0x08） |
| 大小标注 | 右侧，十进制 + 类型（如 "8B (int64)"） |
| 字段名 | 矩形内居中 |
| 类型信息 | 字段名下方，小字 |

#### 尺寸参考
```
基准宽度: 400px（代表结构体最大对齐单元）
每行高度: 30px（数据字段）/ 20px（padding）
左侧偏移列宽: 60px
右侧大小列宽: 100px
行间距: 4px
```

#### viewBox 计算
```
viewBox宽度 = 左边距(30) + 偏移列(60) + 间距(10) + 基准宽度(400) + 间距(10) + 大小列(100) + 右边距(30) = 640
viewBox高度 = 上边距(20) + 标题区(40) + (字段数 × 34) + 底部总计(30) + 下边距(20)
```

#### 示例结构

```xml
<!-- 数据来源: include/storage/buf_internals.h -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 660 320">
  <!-- 偏移列 -->
  <text x="50" y="75" text-anchor="end" font-size="10" font-family="monospace" fill="#64748b">0x00</text>
  <!-- 字段条 -->
  <rect x="70" y="62" width="400" height="28" fill="#dbeafe" stroke="#3b82f6" stroke-width="1" rx="3"/>
  <text x="270" y="80" text-anchor="middle" font-size="11" font-family="monospace" fill="#1e293b">buf_id</text>
  <!-- 大小标注 -->
  <text x="480" y="80" font-size="9" font-family="sans-serif" fill="#64748b">4B (int)</text>

  <!-- padding -->
  <rect x="70" y="94" width="400" height="20" fill="#f1f5f9" stroke="#e2e8f0" stroke-width="1" stroke-dasharray="4,2" rx="3"/>
  <text x="270" y="108" text-anchor="middle" font-size="9" font-family="sans-serif" fill="#94a3b8">(padding)</text>
</svg>
```

### 5. 调用关系图（新增）

用于可视化函数间的调用依赖关系。

#### 布局原则
- **有向图**：节点 = 函数，边 = 调用方向
- **分层排列**：按调用深度分层（调用者在上方）
- **同层间距**：≥80px
- **层间距**：≥60px

#### 节点样式

| 调用深度 | 填充色 | 说明 |
|---------|--------|------|
| Depth 0 (入口) | `#dbeafe` | 分析目标函数 |
| Depth 1 | `#dcfce7` | 直接调用 |
| Depth 2 | `#fef3c7` | 间接调用 |
| Depth 3+ | `#f1f5f9` | 远端调用 |

#### 节点尺寸
```
宽度: max(120, 函数名字符数 × 8 + 20)
高度: 32px
圆角: rx=6
```

#### 连线样式
- 实线箭头：直接函数调用
- 虚线箭头：函数指针/间接调用
- 标注：调用次数或条件（可选）

#### viewBox 计算
```
viewBox宽度 = 左边距(30) + (max_layer_nodes × node_width) + (gaps × 80) + 右边距(30)
viewBox高度 = 上边距(20) + 标题区(40) + (层数 × 92) + 下边距(20)
```

### 6. 状态机图

- 圆角矩形表示状态
- 弧线箭头表示转换
- 箭头旁标注触发条件
- 初始状态用实心圆点表示
- 终止状态用双圆圈表示

## 生成后自检清单

1. [ ] 所有文本在 viewBox 内（无截断）
2. [ ] 无 `<style>` 块，全部内联样式
3. [ ] 技术数值已验证源码（位宽、最大值、偏移量等）
4. [ ] 右侧元素距 viewBox 右边界 ≥30px
5. [ ] 底部元素距 viewBox 下边界 ≥20px
6. [ ] 流程图节点无交错重叠
7. [ ] 连线未穿过其他节点
8. [ ] 数据来源已用 XML 注释标注
9. [ ] 字段边界与位号/偏移标注对齐（位字段图/内存布局图）
10. [ ] 调用关系图中模块着色正确
11. [ ] 跨分支碰撞扫描通过：不同分支节点 x 有交集时 y 间距 ≥ 40px
12. [ ] 连线绕行验证：需要穿越其他分支区域时已用折线 path 绕开
